RL 基础：一切从闭环开始

强化学习可以先压缩成一句话：策略根据状态选动作，环境返回奖励和新状态，算法用这些经验提升长期回报。

同一张图可以解释机器人和大模型：机器人环境是真实世界或仿真器，LLM 环境可以是用户、奖励模型、验证器或偏好数据。

MDP

标准数学模型：状态空间、动作空间、转移、奖励和折扣因子。

\mathcal{M} = (\mathcal{S}, \mathcal{A}, P, R, \gamma)

Return

从当前时刻往后累计的折扣奖励，是策略真正想优化的对象。

G_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k}

V / Q / Advantage

V 评价状态，Q 评价状态-动作，A 衡量某动作比平均水平好多少。

A^\pi(s,a) = Q^\pi(s,a) - V^\pi(s)

Online vs Offline：能不能继续采数据

Online RL 训练时策略会继续和环境交互：做动作、拿奖励、更新，再继续采样。它能主动探索和纠错，但真机成本高、风险高。Offline RL 只用固定数据集训练，不再向环境试错，所以更安全省钱，但最怕数据覆盖不足和 OOD action。

On-policy vs Off-policy：数据来自谁

On-policy 要求更新数据基本来自当前策略，例如 PPO rollout 后马上更新。Off-policy 允许用旧策略、专家、人类或 replay buffer 里的数据，例如 SAC、TD3、DQN。它更省样本，但要处理分布偏移和重要性校正。

轴核心问题含义典型例子机器人 / VLA 启发
Online RL训练时还能采新数据吗能，策略边和环境交互边更新。PPO 在线 rollout；SAC 边采边放 replay buffer适合仿真和可控真机闭环，但要管安全、成本和探索风险。
Offline RL训练时还能采新数据吗不能，只能用固定数据集。CQL、IQL、TD3+BC、BC适合昂贵真机数据和历史遥操作数据；核心难点是数据覆盖和保守更新。
On-policy更新数据必须来自当前策略吗基本必须，旧数据很快失效。PPO、A2C、TRPO更新稳定、逻辑清楚，但样本效率低，真机上通常要很谨慎。
Off-policy更新数据必须来自当前策略吗不必须，可以复用旧数据或别的策略数据。SAC、TD3、DQN、RLPD更适合真机，因为能复用 replay、示教和干预数据，但要防止分布偏移。

不要把 online 等同于 on-policy：SAC 常见是 online + off-policy。也不要把 off-policy 等同于 offline：off-policy 可以一边采新数据，一边复用旧数据。

公式速查

这一节不是为了背公式，而是为了知道每个算法在“优化什么”。公式块可以点击复制。

Policy Gradient

nabla_theta J(theta) =
E_{pi_theta}[ nabla_theta log pi_theta(a_t|s_t) A_t ]

KL Regularization

D_KL(pi_theta || pi_ref) =
E_{y~pi_theta}[ log pi_theta(y|x) - log pi_ref(y|x) ]

Reward Model

P(y_w > y_l | x) =
sigma(r_phi(x,y_w) - r_phi(x,y_l))

Advantage Normalization

A_hat = (A - mu_A) / (sigma_A + epsilon)

IQL Policy Update

max_pi E_{(s,a)~D}[ exp(beta(Q(s,a)-V(s))) log pi(a|s) ]

AWAC

max_pi E_D[ exp(A(s,a)/lambda) log pi(a|s) ]

学习路线

建议先建立统一抽象，再按你的目标选择路线。机器人方向更重视连续控制、样本效率和安全；大模型方向更重视偏好、奖励、KL 约束和长序列稳定性。

路线 A：机器人 RL 优先

RL 基础循环 -> Value / Q / Advantage -> SAC 连续控制 -> Offline RL -> PPO -> VLA 真机闭环 -> LLM RLHF。

路线 B：LLM RL 优先

RL 基础循环 -> LLM 的状态/动作/奖励 -> PPO-RLHF -> DPO -> GRPO -> Offline RL 分布问题 -> VLA。

路线 C：工程实现优先

Behavior Cloning -> Offline Dataset -> SAC/PPO baseline -> Reward/Preference 数据 -> DPO/GRPO -> VLA 数据闭环。

统一类比

机器人状态是图像、本体状态和环境反馈；LLM 状态是 prompt 与已生成上下文。机器人动作是控制命令；LLM 动作是 token。奖励可以是任务成功、人工偏好、规则验证器或 learned critic。

两组容易混的轴

Online/offline 说训练时能不能继续从环境采新数据；on/off-policy 说更新时能不能使用旧策略或其他策略的数据。SAC 常见是 online + off-policy；PPO 通常是 online + on-policy；CQL/IQL 属于 offline RL。

PPO：稳定小步更新

PPO 是机器人仿真和经典 RLHF 都绕不开的算法。它的核心不是“最强”，而是“别一步迈太大”：通过新旧策略概率比和 clipping，限制策略更新幅度。

LCLIP(θ) = Et[ min( rt(θ) At, clip(rt(θ), 1-ε, 1+ε) At ) ]
rt(θ) = πθ(at|st)πθold(at|st) 
L^{CLIP}(\theta) =
E_t[
  min(
    r_t(\theta) A_t,
    clip(r_t(\theta), 1-\epsilon, 1+\epsilon) A_t
  )
]

r_t(\theta) = \pi_\theta(a_t|s_t) / \pi_{\theta_{\mathrm{old}}}(a_t|s_t)
当前策略采样-> 奖励 / value 估计-> 计算 advantage-> clip 更新-> 检查 KL 和成功率

机器人里怎么用

常见于 Isaac Gym 等大规模仿真：并行 rollout，算 GAE，用 PPO 更新，再通过 domain randomization 迁移到真机。真机上从零 PPO 通常样本太贵。

LLM 里怎么用

传统 RLHF 用 reward model 给回答打分，再用 PPO 更新语言模型，同时用 KL penalty 让模型不要偏离 SFT/reference model 太远。

DPO：直接学习偏好

DPO 不是让模型“只模仿好答案”，而是让模型学会：同一个 prompt 下，chosen 应该比 rejected 更可能出现，并且这个偏好是相对 reference model 来衡量的。

LDPO(θ) = -E[ log σ( β( log πθ(yw|x) - log πref(yw|x) - log πθ(yl|x) + log πref(yl|x) ) ) ] 
L_{DPO}(\theta) =
- E [
  log sigma(
    beta [
      log pi_theta(y_w|x) - log pi_ref(y_w|x)
      - log pi_theta(y_l|x) + log pi_ref(y_l|x)
    ]
  )
]

直觉

把偏好对看成二分类：模型应该判断 chosen 胜过 rejected。DPO 把这个偏好直接写进 policy loss，不需要显式训练 reward model。

流程

准备 (prompt, chosen, rejected) -> 固定 reference -> 计算四个 logprob -> 更新 policy -> 检查长度、KL 和偏好胜率。

Off-policy 怎么办

DPO 本来常用离线偏好数据；越 off-policy，越要过滤低 reference logprob 样本、调保守 β、做 iterative DPO 或在线刷新候选。

GRPO：用组内相对奖励替代 value model

GRPO 对同一个 prompt 采样多个回答，用组内均值和标准差构造 advantage。它非常适合数学、代码、规则可验证任务，因为 reward 可以来自验证器。

Ai = ri - mean(r1, ..., rG)std(r1, ..., rG) + ε
下面写成最小化 loss；若写成最大化 objective，符号会整体反过来。 
A_i = (r_i - mean(r_1, ..., r_G)) / (std(r_1, ..., r_G) + epsilon)

Loss_{GRPO} =
- E_{i,t} [
  min(rho_{i,t} A_i, clip(rho_{i,t}, 1-epsilon, 1+epsilon) A_i)
  - beta KL(pi_theta || pi_ref)
]

如果一组 reward 全一样，比如全对或全错，那么 std=0，advantage 没有区分度。工程上先加 epsilon 防 NaN，但这组仍然没有学习信号。常见处理是跳过该 group、重新采样、增大 group size、提高采样温度、加入过程奖励或做课程学习。

LLM RL 算法族

大模型 RL 的主要问题不是连续控制，而是长序列 credit assignment、reward hacking、KL 控制、长度偏置和采样效率。

on-policy需要 critic

PPO-RLHF

Reward model 打分，value model 估计 advantage，PPO 更新 policy，KL 限制偏离 reference。成熟但工程重。

offline无需 RM

DPO

直接用 chosen/rejected 偏好对优化 policy。便宜稳定，但不能主动探索新回答。

on-policy无需 value

GRPO

同 prompt 采样多条回答，用组内相对奖励做 advantage。适合数学、代码、验证器奖励。

on-policybaseline

RLOO

同 prompt 采样 K 条，第 i 条用其他 K-1 条平均奖励作 baseline，简单地降低 REINFORCE 方差。

轻量 RL序列奖励

ReMax / REINFORCE++

用完整回答奖励做 policy gradient，再加 baseline、KL、归一化、clip 等技巧。比 PPO 轻，但方差要处理。

GRPO 增强reasoning

DAPO

围绕 GRPO 做动态采样、非对称 clip、token-level loss 和长度处理，目标是提升大规模推理 RL 稳定性。

sequence-level长 CoT

GSPO

把 ratio/clipping 从 token 级提升到 sequence 级，缓解长序列或 MoE 训练中的局部不稳定。

缩写不统一看论文定义

TPPO

常见语境可能是 Token-level PPO，也可能是 Turn-level PPO。前者做更细 token 信用分配，后者面向多轮 agent。

偏好优化DPO 家族

IPO / KTO / ORPO / SimPO

DPO 后续偏好优化族：有的修正目标，有的支持非成对反馈，有的合并 SFT 与 preference loss。

机器人 RL 算法族

机器人强化学习的关键词是样本效率、安全和动作表达。多数真实项目不是 RL from scratch，而是 BC/示教起步，再通过 offline/off-policy/residual RL 小心改进。

离线模仿学习

BC

把策略学习当监督学习：给定观测拟合专家动作。高效，但遇到专家没覆盖的状态容易崩。

L_{BC} = - E_{(s,a)~D}[ log pi_theta(a|s) ]
交互式模仿纠偏

DAgger

让当前策略跑起来，再请专家标注它实际遇到的状态，逐轮减少 covariate shift。

off-policy连续控制

SAC

最大化奖励同时最大化 entropy，利用 replay buffer 提升样本效率。真实机械臂连续控制常用强基线。

J(pi) = E[ sum_t gamma^t ( r(s_t,a_t) + alpha H(pi(.|s_t)) ) ]
off-policy确定性策略

TD3 / DDPG

连续动作 actor-critic。TD3 用双 Q、延迟 actor 更新和 target smoothing 修复 DDPG 的不稳定。

稀疏奖励目标条件

HER

失败轨迹也能用：把实际达到的状态当作新目标重标，适合 reaching、pushing、relocation。

真机友好残差动作

Residual RL

已有控制器或 BC/VLA 输出基础动作，RL 只学小残差：更安全，也更适合接触细节修正。

a_t = a_t^{base} + Delta a_t^{RL}
离线+在线示教启动

RLPD

把 prior data 和 online data 混进同一个训练闭环，用示教数据降低初期探索成本。

加权模仿离线到在线

AWAC / AWR

用 advantage 给数据动作加权，好的动作更大权重，适合从 demo 平滑过渡到在线改进。

安全约束真机

CPO / Lagrangian RL

把安全约束显式写进优化目标，例如碰撞率、力矩、越界次数。真机里常和 shield/CBF 搭配。

Offline RL：不要轻信数据外动作

机器人里大量数据来自示教和日志。Offline RL 的痛点是：策略可能选择数据集中没出现过的动作，而 Q 网络却错误地给它高分。

CQL

压低数据外动作 Q 值，宁可保守，也不要让策略被错误高估吸引到危险动作。

L_{CQL} = L_{Bellman} + alpha( E_{a~pi}[Q(s,a)] - E_{a~D}[Q(s,a)] )

IQL

避免显式查询 OOD 动作，通过 expectile value 和 advantage-weighted BC 在数据支持范围内变好。

TD3+BC

TD3 actor 目标加入 BC 正则，让策略既追求高 Q，又贴近数据集动作，是简单强离线基线。

Diffusion Policy 与 VLA

现代机器人学习越来越常见的路线是：先用大量示教训练能生成动作序列的策略，再用 RL 或 critic 做后训练。VLA 则把视觉、语言和动作统一起来。

模仿学习action chunk

Diffusion Policy

从噪声逐步去噪生成未来一段动作。适合多解、长时序、视觉操作任务，比如开抽屉、擦拭、插入。

L = E_{k, eps, a_0}[ || eps - eps_theta(a_k, k, o) ||^2 ]
RL 微调PPO 风格

DPPO

把扩散策略的动作序列分布拿来做 PPO 风格优化。优势是继承强模仿先验，风险是更新过大会破坏先验。

a_{t:t+k} = \pi_\theta(o_t, x, q_t)

o_t: 视觉观测
x: 语言指令
q_t: 机器人本体状态
a_{t:t+k}: 未来一段动作序列

VLA 真机 RL / 在线后训练

真机 RL 的关键不是让策略乱试，而是建立一个安全、可回滚、能复用数据的闭环。通常路线是：强 BC 初始化 -> 安全 rollout -> replay/off-policy 更新 -> critic/reward 改进 -> 小批量上线。

VLA 真实机器人 RL：先把策略放进可执行分布，再用真实失败/成功数据、critic、KL/BC 正则做小步在线后训练。

  1. 先用预训练 VLA + 任务数据 SFT/BC，让策略基本能做。
  2. 动作空间优先选末端位姿增量、速度指令或 action chunk，不要一开始直接输出底层力矩。
  3. 设置工作空间、速度、力/扭矩、夹爪压力、安全停止和人类接管。
  4. 用真实 rollout 收集成功、失败、人工介入和安全事件。
  5. 训练 learned reward / VLA-Critic，或用规则成功判定给 outcome reward。
  6. 用 replay/off-policy/residual RL 更新，并加入 KL/BC 正则限制策略漂移。
  7. 小批量上线，若安全事件上升或成功率下降，立即回滚。

  RL objective

+ lambda_bc * BC_loss(a_new, a_expert)
+ lambda_safety * unsafe_penalty
  • 工作空间边界、急停、软件 stop 可用
  • 动作限幅、速度限制、力阈值已启用
  • 相机标定、时间同步、任务初始状态稳定
  • 成功/失败判定规则已实现
  • Replay buffer 存 obs/action/reward/done/success/intervention
  • 人类介入轨迹可作为纠错 BC 数据
  • 有稳定参考策略 π_ref，可随时回滚
  • Reward model / VLA-Critic 有独立验证集
  • 上线前做离线动作回放检查
  • 评估包含成功率、安全事件、介入次数和泛化场景
